# How to use

The main colors can be reference by a matris about meaning of the content (primary, secondary, accent) and target of the usage (background, container, default). Each
of those colors also provides a color which is intendet to be used on top the this color with the `on` prefix. Besides those colors, there are also other colors for 
special purposes.

## Meaning of Content

- Primary

  Use the primary color for content and controls which is the most importand on the current view. 

- Secondary

  Use the secondary color for content and controls wich supplement the primary content and controls. 

- Accent

  Use the accent color if you want to set an accent to a specific item on your view or to catch the focus of the user for special items on the view.

## Target of the usage

- Background

  Use the background type of the color if you want to use it as background.

- Container

  Use the container type of the color if you want to use it as a container for content, for instance a card, table or other controls which can contain content.

- default

  Use the default type of the color for small items which are intented to get the focus of the user, for instance buttons, checkboxes, etc.


## Other Colors

- Shade

  Use this color to shade the background at the place you use this color

- Tint

  Use this color to tint the background at the place you use this color

- Succsss

  Use this color for success notifications and hints

- Error

  Use this color for error notifications and hints

- Background

  This color is automatically used as the main background color for the entire application

- Background Variant

  This color is used for containers which need to be some kind of a layer above the background

- Outline

  This color is used for outlining content

- Hover

  This color is used to shade the background on hovering an item.
